# TCP vs UDP

- 是否面向连接
  - TCP 面向连接、三次握手四次挥手
  - UDP 无连接
- 可靠性
  - TCP 可靠传输，ACK（确认应答）、重传、流量控制、拥塞控制
  - UDP 不可靠，尽最大努力交付
- 是否有状态
  - TCP 有状态，维护连接信息
  - UDP 无状态，开销小
- 传输效率
  - TCP 开销大，效率低
  - UDP 开销小，速度快
- 传输形式
  - TCP 面向字节流
  - UDP 面向报文
- 首部大小
  - TCP 最少20字节
  - UDP 固定8字节
- 广播/ 多播
  - TCP 仅支持单播
  - UDP 支持单播、多播、广播
 
TCP：面向连接、可靠、有序、拥塞控制、流量控制、滑动窗口、慢启动
UDP：无连接、不可靠、无序、不重传、速度快、简单
TCP 可靠但复杂，UDP 简单但不可靠。

网络极好、无丢包、无拥塞时：TCP 和 UDP 速度几乎一样
只有丢包、拥塞、抖动时，TCP 才会变慢（因为要重传、降速）

什么情况下 TCP 不会比 UDP 慢？

满足以下条件时：

网络质量极好（无丢包、无延迟、带宽充足）
滑动窗口拉满（rwnd、cwnd 都很大）
没有拥塞、没有丢包
不需要实时性（文件下载这种场景）
此时 TCP 速度 = 物理带宽极限，和 UDP 一样快。

TCP 不一定比 UDP 慢，网络好时速度一样。
文件传输必须用 TCP，因为要可靠。
TCP 速度由滑动窗口 min (rwnd, cwnd) 决定，
下载快慢取决于网络是否拥塞、是否丢包、窗口大小。
