# ACID

## Atomicity
- 原子性 
  - 事务中的操作要么全部 commit 成功
  - 要么全部在失败中 rollback 到开始前的状态
  - 不会结束在中间某个环节

## Consistency
- 一致性
  - 事务执行前后，数据库都必须处于合法一致的状态
  - 如约束、外键、触发器不会被破坏

## Isolation
- 隔离性
  - **并发事务**之间相互隔离，彼此的中间结果不可见
  - **隔离级别**决定可见度

## Durability
- 持久性
  - 事务一旦 commit，其结果会被永久保存
  - 即使数据库崩溃、系统故障也不会丢失
 

原子性（A）：事务不可分割，要么全成功，要么全失败回滚。
一致性（C）：执行前后数据整体合法、总数不变、业务约束满足。
隔离性（I）：并发事务之间互不干扰，各自独立。
持久性（D）：提交后数据永久生效，宕机也不丢。

AID 是手段，C 是目的！只有保证原子、隔离、持久，才能最终保证一致性。
