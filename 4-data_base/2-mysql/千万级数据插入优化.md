# 千万级大数据插入优化

核心思路：

减少交互次数 + 关闭冗余校验 + 批量提交 + 硬件 / 配置优化


- 批量插入

不要一行行执行 insert，一次性插入多值

- 关闭自动提交

数据库默认每执行一条sql就提交一次事务，巨量IO操作

优化方法：开启手动事务，最后统一提交。

```sql
START TRANSACTION;  -- 开始事务

-- 这里执行 1000 条批量插入
INSERT INTO t VALUES(...);
INSERT INTO t VALUES(...);

COMMIT;  -- 一次性提交
```

- 临时关闭索引、外键约束（插入完再开启）

索引和外键会每次插入一行就重新计算

导入后重建索引

```sql
-- 插入前
SET FOREIGN_KEY_CHECKS=0;
ALTER TABLE t DISABLE KEYS;

-- 执行大批量插入

-- 插入后
ALTER TABLE t ENABLE KEYS;
SET FOREIGN_KEY_CHECKS=1;
```

- 使用数据库原生导入工具

适用于从excel或csv文件导入数据

- 代码层优化

1. 使用预处理语句
2. 使用批量执行api
3. 每500-1000条提交一次事务

- 数据库配置优化

```sql
innodb_flush_log_at_trx_commit = 2  # 刷盘策略（牺牲一点安全性换极快速度）
innodb_buffer_pool_size = 70%内存    # 加大缓存
bulk_insert_buffer_size = 64M
```
