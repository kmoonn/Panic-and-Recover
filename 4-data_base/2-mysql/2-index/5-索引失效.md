# 索引失效

SELECT * 不会直接让索引失效，但会导致无法使用覆盖索引而必须回表；
当结果集较大时，优化器可能选择全表扫描，看起来像索引失效。

## 失效原因

违反最左前缀原则，联合索引没有从最左列开始匹配。

索引列上使用函数、计算、类型转换，破坏索引有序性 → 索引失效。

LIKE 以 % 开头，LIKE '%abc' 无法使用索引；LIKE 'abc%' 可以走索引。

OR 连接的列有一个没有索引，OR 前后必须都有索引，否则整个条件索引失效。

IN / NOT IN 取值范围过大，优化器认为回表成本太高，转而全表扫描。

隐式类型转换，如 varchar 列用数字查询，导致索引失效。

最左不匹配、索引列运算、左模糊、OR 缺索引、IN 范围大、隐式转换，都会导致索引失效。

order by 乱搞 导致无法用索引排序

优化器觉得全表更快（数据太少）
