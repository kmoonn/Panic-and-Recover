# IoC

控制反转

一种设计思想

传统开发：程序员 new 对象，主动创建、管理依赖（主动控制）
Spring IOC：把对象的创建、依赖管理、生命周期全权交给 Spring 容器，程序员直接注入使用（控制权反转给容器）

IOC 是思想，DI 是实现 IOC 的核心手段，容器自动给 Bean 注入依赖属性。

将对象的创建、依赖关系的绑定、生命周期管理等原本由程序员在代码中主动控制的职责，交由 Spring 容器（BeanFactory 或 ApplicationContext）统一接管和自动化处理

其典型实现方式是依赖注入（DI，Dependency Injection），即容器在运行时自动将依赖对象注入到目标 Bean 中，而非由 Bean 自己 new 或查找依赖

IOC 的本质不是技术而是设计原则——控制权从应用代码“反转”给框架容器
