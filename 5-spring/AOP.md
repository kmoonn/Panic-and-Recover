# AOP

面向切面编程

一种横向抽取通用公共逻辑的编程思想，在不修改原有业务代码的前提下，对方法进行增强。

OOP 是纵向继承、横向组合；
AOP 是横向切片，统一拦截增强。

## 使用场景

1. 统一日志记录

拦截 Controller / Service 所有接口
自动记录：请求参数、响应结果、请求耗时、请求 IP、操作人
不用每个接口手动打 log.info

2. 统一权限校验、登录拦截

切面拦截需要登录 / 鉴权的接口
统一校验 Token、权限角色，无权限直接拦截返回
避免每个接口写重复的 token==null 判断

3. 统一事务管理
4. 全局异常统一处理 + 告警
5. 接口限流、防重复提交
6. 操作日志 / 审计日志
7. 性能监控、慢接口查询
8. 数据脱敏、字段加解密
