# 网络分层模型

OSI 七层模型

- 应用层
    - 为用户提供服务
- 表示层
    - 数据处理（编解码、加解密、压缩解压）
- 会话层
    - 管理应用程序间的会话（建立、维护、重连）
- 传输层
    - 为两台主机进程之间的通信提供数据传输服务
- 网络层
    - 路由、寻址
- 数据链路层
    - 帧编码、误差纠正控制
- 物理层、
    - 透明地传输比特流

TCP/IP 四层模型

- 应用层
- 传输层
- 网络层
- 网络接口层

为什么要分层

- 各层之间相互独立，只需要调用下层提供好的功能
    - 例如开发时的分成（controller层提供接口、service层业务操作、repository层数据库交互）
- 提高灵活性和可替换性，每一层都可以使用最合适的技术来实现，每一层都可以根据需要进行修改或替换
    - 开发系统时的原则：高内聚低耦合
- 简化问题，将复杂的网络系统变得易于设计，实现和标准化
