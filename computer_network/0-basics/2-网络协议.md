# 网络协议

应用层

- HTTP
    - 基于tcp
    - 传输超文本和多媒体内容
    - web浏览器与web服务器之间通信
    - 浏览器网页加载
- SMPT
    - 简单邮件发送协议
    - 基于tcp
    - 发送电子邮件
- POP3/IMAP
    - 基于tcp
    - 接收电子邮件
    - IMAP比POP3更新，支持邮件搜索、标记、分类、归档，多设备同步邮件状态
- FTP
    - 基于tcp
    - 在网络计算机间传输文件
    - 可屏蔽不同操作系统和文件存储差异
    - 不安全，传输过程不进行数据加密
        - SFTP是安全的FTP
- Telnet
    - 远程登录协议
    - 基于tcp
    - 通过一个终端登录到其他服务器
    - 明文发送数据
        - 安全替代：SSH
- SSH
    - 安全的网络传输协议
    - 基于tcp
    - 通过加密和认证机制实现安全访问和文件传输
- RTP
    - 实时传输协议
    - 通常基于UDP，但也支持TCP
    - 提供端到端的实时数据传输
    - 不包含资源预留（提前申请一段固定带宽）、不保证实时传输质量
        - WebRTC 可以实现
- DNS
    - 通常基于UDP
    - 端口号53
    - 域名和ip映射
    - 当响应数据过大（超过512字节，udp限制512字节）或进行主从区域传送时会改用tcp

传输层

- TCP
    - 传输控制协议
    - 面向连接
    - 可靠
- UDP
    - 用户数据报协议
    - 无连接
    - 尽最大努力，不保证可靠

网络层

- IP
    - 网际协议
    - 定义数据包格式、主机寻址（IP地址）、数据包路由转发
    - IPv4 32位
    - IPv6 128位
    - 不可靠、无连接、尽力交付
- ARP
    - 地址解析协议
    - 网络层与数据链路层地址间转换
    - 逻辑地址与物理地址转换
    - IP 转 MAC 地址映射
- ICMP
    - 互联网控制报文协议
    - 传输网络状态和错误消息
    - 网络诊断、故障排除
    - Ping 工具
        - 测试网络连通性
- NAT
    - 网络地址转换协议
    - 内部网络到外部网络的地址转换
    - 内网私有IP 与 公网公有IP 互相转换
    - ipv4不够用，多台内网设备公用一个公网ip上网
- OSPF
    - 开放式最短路径优先
    - 内部网关协议 IGP
    - 一种动态路由协议
    - 基于链路状态算法，考虑链路带宽、延迟等因素来选择最佳路径
- RIP
    - 路由信息协议
    - 内部网关协议 IGP
    - 一种动态路由协议
    - 基于距离向量算法，使用固定的跳数作为度量标准，选择跳数最少的路径作为最佳路径
- BGP
    - 边界网关协议
    - 外部网关协议 RGP
    - 在不同的自治系统之间交换路由信息
    - 路径矢量协议
