# HTTP/1.1 范围请求和断点续传

HTTP/1.1 通过 Range 请求头、Content-Range 响应头、206 状态码实现范围请求，从而支持断点续传、多线程下载和视频进度拖拽。

## 核心思想

**客户端告诉服务器：我只要文件的某一段字节，不要整个文件。**

服务器支持的话，就只返回那一段，从而实现：

- 断点续传
- 多线程下载
- 视频拖拽进度条

## 关键请求头/响应头

- 客户端请求头：Range
- 服务器响应头
  - 是否支持范围请求 Accept-Range：bytes
  - Content-Range：bytes start-end

## 断点续传

1. 客户端第一次下载，断开或暂停
2. 客户端再次下载，发送range头，告诉服务器从哪个字节开始
   - 单线程时，只需要计算本地已下载字节数
   - 多线程/分片下载时，客户端会提前把文件切成若干段，每个线程下载一段，则需要分别计算start，end
3. 服务器检查文件大小，返回206和content-range
4. 客户端把新收到的片段追加到本地文件
5. 重复只到下载完成
