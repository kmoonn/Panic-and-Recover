# TCP 传输控制协议

一种面向连接的、可靠的传输层协议。

可靠：按序交付、差错检测、丢包重传、流量控制、拥塞控制。

## 三次握手

- 第一次握手 SYN
  - 客户端 → 服务端：发送 **SYN 包**，随机初始序列号 **seq = x**
  - 客户端状态：**CLOSED → SYN_SENT**
- 第二次握手 SYN + ACK
  - 服务端 → 客户端：回复 **SYN+ACK 包**
    - 自己的随机序列号 seq = y
    - 确认号 **ack = x + 1**
  - 服务端状态：**LISTEN → SYN\_RCVD**
- 第三次握手 ACK
  - 客户端 → 服务端：发送 **ACK 包**
    - 序列号 **seq = x + 1**
    - 确认号 **ack = y + 1**
  - 双方状态：都变为 **ESTABLISHED**
  - 连接建立成功

**一握 SYN 我要连，二握 SYN+ACK 同意连，三握 ACK 连上了，双方 ESTABLISHED。**

三次握手是为了保证通信双方的「发送能力」和「接收能力」都正常，并且同步双方初始序列号 ISN（为可靠传输做准备）。

如果只使用两次握手，服务端在发送 SYN+ACK 后就认为连接建立，但无法确认客户端是否收到。

若网络中存在延迟的旧 SYN 包，服务端会建立无效连接（与客户端丢弃的SYN建立连接），浪费资源，甚至导致 SYN 洪水攻击。
