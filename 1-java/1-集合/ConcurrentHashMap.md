# ConcurrentHashMap

## 数据结构

- JDK 1.7
  - 数组 + 链表
- JDK 1.8
  - 数组 + 链表 + 红黑树

哈希表

## 特点

- **线程安全**

通过分段锁机制实现线程安全：即对每个Segment独立加锁，小数组HashEntry用于存储键值对数据。

---

JDK 1.8 核心安全原理

1. 取消分段锁，采用 CAS + synchronized 锁住单个桶位
2. 读操作无锁，大量并发读不阻塞
3. 写操作只锁当前hash桶，不同的key并发写入互不影响
4. 天然支持并发遍历，不需要手动加锁。


---
1. 不保证复合操作原子性，不能先get再put
2. 不能存null键 / 值，会空指针异常
