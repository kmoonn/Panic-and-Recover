# 索引底层数据结构

InnoDB、MyISAM 默认索引：B+Tree

## 哈希索引

不支持范围查询： 哈希值是无序的，无法处理 >,<,BETWEEN 等范围扫描。
不支持排序： 无法利用 ORDER BY 进行优化。
不支持联合索引： 必须使用全部列才能匹配，不支持部分索引键。
哈希冲突： 极端情况下哈希冲突严重会降低性能。 

哈希索引只适合精确等值查询，无法满足数据库通用场景。

哈希索引只适合等值查询，不支持范围与排序

## B+ 树

I/O 更少非叶子节点不存数据，只存键，树更 “矮胖”，扇出更大。
查询性能更稳定所有查询都必须走到叶子节点，路径长度一致。
范围查询极快叶子节点用双向链表连接，范围查询只需顺序遍历链表，无需回溯上层。

B + 树比 B 树 I/O 更少、查询更稳定、范围查询更快
