# 磁盘调度算法

1. 先来先服务（FCFS）
按请求到达顺序处理。
优点：简单、无饥饿。
缺点：平均寻道时间长，效率低。

2. 最短寻道时间优先（SSTF）
优先处理离当前磁头最近的请求。
优点：寻道时间短，效率高。
缺点：会产生饥饿（远处请求长期得不到服务）。

3. 扫描算法（SCAN / 电梯算法）
磁头朝一个方向移动，处理沿途请求，到达边界再反向。
优点：无饥饿，性能较好。
缺点：到达另一端才换向，可能有无效移动。

4. 循环扫描（C-SCAN）
只向一个方向处理请求，到达终点直接快速返回起点再循环。
优点：响应时间更均匀。
缺点：返回时不处理请求。

5. LOOK 算法
SCAN 改进：磁头移动方向无请求就立即换向，不用走到边界。
优点：减少无效移动，效率更高。

6. C-LOOK 算法
C-SCAN 改进：磁头只在有请求的范围内循环，直接返回到最远端请求处。
优点：性能最好，现代 OS 常用。
