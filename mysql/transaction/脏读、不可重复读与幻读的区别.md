# 脏读、不可重复读与幻读的区别

MySQL 服务端允许多个客户端连接，意味着会出现同时处理多个事务的情况。

## 脏读 dirty read

A 事务读取到了 B 事务**修改过尚未提交事务**的数据

B 事务可能出错发生了回滚，导致 A 事务读取的数据其实是无效的（脏数据）。

## 不可重复读 non-repeatable read

一个事务内多次读取同一个数据，前后两次读取到的**数据不一样**。

事务 A 先读取数据 X，然后事务 B 更新了 X 并提交了事务，事务 A 再次读取 X 时发现前后两次读到数据不一样。

## 幻读 phantom read

一个事务内多次执行相同的条件查询，前后两次查询到的**记录数量不一样**。

事务 A 在事务 B 两次相同条件查询中间执行了符合查询条件的插入操作。
