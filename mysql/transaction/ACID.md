# 事务的四大特性 ACID

## Atomicity
- 原子性 
  - 事务中的操作要么全部 commit 成功
  - 要么全部在失败中 rollback 到开始前的状态
  - 不会结束在中间某个环节

## Consistency
- 一致性
  - 事务执行前后，数据库都必须处于合法一致的状态
  - 如约束、外键、触发器不会被破坏

## Isolation
- 隔离性
  - **并发事务**之间相互隔离，彼此的中间结果不可见
  - **隔离级别**决定可见度

## Durability
- 持久性
  - 事务一旦 commit，其结果会被永久保存
  - 即使数据库崩溃、系统故障也不会丢失
