# 硬链接、软链接

硬链接：和原文件共用同一个 inode，是文件的另一个入口。

软链接：有独立 inode，存储的是原文件路径，类似 Windows 快捷方式。

删除源文件不影响硬链接，只有所有链接都删完文件才真正删除。

源文件删除后，软链接变成死链接（失效）。

硬链接：不能跨文件系统、不能链接目录。
软链接：可以跨文件系统、可以链接目录。

硬链接：ln 源文件 目标
软链接：ln -s 源文件 目标

每个文件系统有独立的 inode 表，inode 只在本文件系统内唯一。
跨文件系统会导致 inode 冲突，因此硬链接不能跨文件系统。

硬链接共享 inode，不能跨区、不能目录，删源不影响；
软链接是快捷方式，独立 inode，可跨区可目录，删源即失效。
