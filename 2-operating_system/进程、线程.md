# 进程、线程

进程 = 工厂
线程 = 工厂里的工人

进程是资源分配的基本单位，有独立地址空间，隔离性好、开销大；
线程是 CPU 调度的基本单位，共享进程资源，轻量、切换快、但一个线程挂掉可能导致整个进程崩溃。

## 线程同步

- 互斥锁

独占访问，同一时刻只允许一个线程进入临界区。

- 读写锁

读共享、写独占。
适合读多写少场景，提高并发效率。

- 信号量

控制同一时刻最多允许多少个线程访问资源。

并发限流

- 屏障

等待所有线程都到达某个点后再一起继续执行。

线程协同

- 等待/通知机制

线程等待某个条件满足，其他线程通过通知唤醒。

等待唤醒

## 进程状态

5 状态模型

创建状态（New）进程正在被创建，还没进入就绪队列。

就绪状态（Ready）进程已获得除 CPU 以外的所有资源，只要分到时间片就能立刻运行。

运行状态（Running）进程正在 CPU 上执行。单核 CPU 同一时刻只有一个进程运行。

阻塞状态（Waiting/Blocked）进程等待某事件（I/O、信号、锁），即使 CPU 空闲也不能运行。

终止状态（Terminated）进程执行完毕或被终止，资源正在回收。
