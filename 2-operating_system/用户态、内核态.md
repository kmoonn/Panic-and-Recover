# 用户态、内核态

用户态权限低，只能跑应用代码；内核态权限高，可操作所有硬件资源。
应用需要底层操作时通过系统调用陷入内核，完成后返回用户态。
由于切换开销大，应尽量减少用户态与内核态的切换次数以提升性能。

- 用户态

应用程序运行的状态，权限低，只能访问用户空间，不能直接操作硬件。需要操作系统服务时，必须通过系统调用进入内核态。

- 内核态

操作系统内核运行的状态，权限极高，可以访问所有内存、硬件、驱动，执行特权指令。


## 系统调用

需要保存 / 恢复用户进程上下文
需要切换页表、堆栈
需要进行权限检查、特权级转换
会污染 CPU 缓存（Cache），导致性能下降

系统调用越少，性能越好。

分用户态和内核态是为了安全、稳定和资源隔离。
内核态执行特权指令，用户态受限运行，
防止应用程序破坏系统，保证操作系统稳定可靠。

## 切换

- 系统调用

主动触发。应用程序需要访问硬件 / 内核服务（read、write、send 等），主动进入内核。

- 中断

被动触发，来自外部硬件。如网卡、磁盘、时钟完成操作，发中断信号给 CPU，切内核态处理。

- 异常

被动触发，来自当前指令执行错误。如除零、非法内存访问、缺页异常，CPU 自动切内核态处理。

中断来自外部硬件，异常来自当前指令本身。
